# Session log — 2026-05-04 18:00

## Original request

Continuation of the Weather-SoC redesign after PR-WSOC-TABLE-1 shipped
the read-only v1 widget. Operator deployed v1, observed:

1. The table is on the Detail page but read-only — no controls. Make
   the cells editable.
2. The column headers (`exp/bat/dis/ext`) should have clickable hints.
3. The 6 boundary knobs (`weathersoc.threshold.energy.{low/ok/high/
   too-much/very-sunny}` + `weathersoc.threshold.winter-temperature`)
   should be hidden from the flat Knobs table and incorporated as
   controls in the Weather-SoC widget.
4. Move the widget from the Detail tab to the Control tab — Detail is
   purely informative.

Two corrections after the first plan iteration:

5. Use a popup modal for cell edits (not inline) so defaults and revert
   controls are visible.
6. `bat` defaults should be 100 across all 12 cells. The current
   table's 90 in Low.cold / Dim.warm / Dim.cold is operator-non-grata
   ("during cold days we need heat pump/hot water; charge to 100").

After confirming the design, the operator chose: 48 flat HA-
addressable knobs (not just internal addressability — each cell field
must be editable from HA), inline boundary edits, popup cell edits,
header hints as listed.

## Milestone landed this session

### M-WSOC-EDIT / PR-WSOC-EDIT-1 (commit `84afa80`)

48 cell knobs as flat HA-addressable knobs via single
`KnobId::WeathersocTableCell { bucket: EnergyBucket, temp: TempCol,
field: CellField }` variant with associated data. Internally one
programmatic match arm per plumbing layer (apply_knob, knob_name,
knob_id_from_name, knob_range, parse_knob_value). Externally HA sees
36 number + 12 switch entities under
`weathersoc.table.<bucket>.<temp>.<field>`.

`EnergyBucket` relocated to a new `crates/core/src/weather_soc_addr.rs`
module alongside new `TempCol` and `CellField` enums + kebab/from_kebab
helpers + `ALL` const slices.

`bat` defaults normalised to 100 across all 12 cells (Low.cold /
Dim.warm / Dim.cold changed from 90 → 100). Three retained
cascade-equivalence tests' `bat` assertions updated with per-test
comments naming "operator preference 2026-05-04 (PR-WSOC-EDIT-1)".
Cascade equivalence is now intentionally relaxed in those three cells.

Cell-edit UX: popup modal via the existing `#entity-modal` with new
`EntityType="weathersoc-cell"` arm. 4 fields per cell (3 numeric + 1
checkbox), each with `[default: X]` label and per-field "Revert"
button + Save / Cancel. Save dispatches only changed fields via the
existing `knob/<name>/set` websocket command channel. Dirty-input
rule: snapshots arriving while modal is open do not clobber focused
or already-changed inputs.

Boundary knobs hidden from `renderKnobs` via new `WIDGET_RENDERED_KNOBS`
set; rendered inline above the cell grid as direct number inputs +
"set" buttons (5 kWh + 1 °C).

Column headers (`exp/bat/dis/ext`) clickable via `<a class="entity-link
mono">` with `entity-type="knob"`, opening the entity-inspector modal
with descriptions sourced from new `descriptions.ts` entries.

Widget moved from Detail tab to Control tab above `<section id="knobs">`.

`knob_name` return type changed `&'static str → String` to support
runtime-formatted cell names. Cascade through `discovery.rs` and
`process.rs::all_knob_publish_payloads`.

Drift guard between Rust `WeatherSocTable::safe_defaults()` and TS
`WEATHER_SOC_DEFAULTS`: a Rust test in
`crates/shell/src/dashboard/convert.rs` writes
`web/test-fixtures/weather-soc-defaults.json` from the safe-defaults
table at test time and asserts equality. The TS-side
`WEATHER_SOC_DEFAULTS` literal mirrors the fixture content. Note:
TS-side assertions are type-checked only — the project has no JS test
runner — so the Rust leg is the load-bearing check.
