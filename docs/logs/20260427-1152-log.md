## Diagnosis

1. **Extended charge never stops.** The legacy Node-RED flow had a
   standalone `00 08 * * *` cron (`legacy/debug/20260421-120500-injects-crons.txt:8`,
   change-node `f93090cc98e44e37`) that injected `chargeMode: Off` at
   the end of the NightExtended window. The Rust port (`evaluate_zappi_mode`)
   reproduced the Boost (02:00–05:00) and NightExtended (05:00–08:00)
   windows but not this standalone 08:00 Off-edge. With
   `charge_car_boost=false` and `charge_car_extended=true`, the controller
   set Fast at 05:00 and after 08:00 returned `Leave`, leaving the Zappi
   in Fast mode forever.
2. **Dashboard schedules section missing Zappi.** `compute_scheduled_actions`
   (`crates/shell/src/dashboard/convert_schedule.rs:48-63`) collected
   four sources (eddi tariff, ESS schedule_0/1, next_full_charge,
   weather_soc) — Zappi entries simply absent, even though the three
   daily edges (02:00 / 05:00 / 08:00) are deterministic.
