## Notes for operator before deployment

- Static analysis only this session; not exercised against the
  live HM051M.U43.
- Phase A operators upgrading to Phase B should run the runbook in
  `tasks.md`'s M-LG-THINQ deploy-runbook subsection to purge zombie
  retained HA discovery entries (6 topic deletions + 1 availability).
- Operator override semantics: dashboard / HA writes to a
  controller-driven knob are overridden within ≤1 tick by
  `HeatPumpControlCore`. The operator's intent IS surfaced via the
  `ActuatedPhase::Pending` publish the operator-knob mirror emits
  before the cores run. This is by design; matches the
  `EssStateOverrideCore` pattern.
- The master heat-pump power slot (`lg_heat_pump_power`) is
  operator-only by design — the controller does NOT drive it. The
  external heat-demand relay owns the contactor.
