  point at the same knob.

## Adversarial review rounds

**Round 1**: 2 minor defects.

- **D01 (minor)** — Re-opening single-knob-edit for the same id
  after a knob-modal transition wedged the modal. Sequence: open
  Low → click sub-header → click Low again → modal stuck on
  read-only knob view. `alreadyOpen` short-circuit keyed only on
  dataset attribute; the body had been overwritten by the
  intermediate `renderKnobBody` call. Fixed: AND the dataset check
  with `bodyEl.querySelector("[data-singleknob-field]")` non-null;
  fall through to full rebuild when input is missing.
- **D02 (minor)** — Defensive enum branch in `saveSingleKnobEdit`
  omitted `knob_name`, would silently produce malformed command if
  a future enum knob were wired to single-knob-edit. Fixed: replaced
  with `throw new Error` — fail loud.

**Round 2**: No new defects; D01 / D02 fixes verified. Reviewer walked
the failure path for D01 (open Low → click `exp` sub-header → click
Low again → confirmed full rebuild fires) and the hypothetical
future-enum case for D02 (confirmed hard-fail beats silent malformed
payload).
