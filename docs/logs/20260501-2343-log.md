## Notes for future work

- **`Actual::unknown` type-side fix** (`since: Option<Instant>`)
  remains deferred. The render-side guard now covers all sensor
  rows. Re-evaluate if the guard ever hides a real bug.
- **Universal actuator retry is now load-bearing**. Every new
  actuator added to the codebase must:
  - Use the gate shape `!changed && !needs_actuation(now,
    retry_threshold)`.
  - Call `mark_commanded` on every actuation push so the retry
    clock advances self-stabilisingly.
- **Typed-sensor-string wire shape is now the convention** for
  string-valued dashboard sensors. New string sensors should not
  synthesise rows in render.ts — populate via `TypedSensorString`.
- **`since_epoch_ms = 0` sentinel for Unknown freshness** is the
  wire-level convention. Distinct from "observed at boot". A future
  render-side change that drops the Unknown-guard would expose
  1970-01-01 dates; re-add the guard then.
- **Cadence pinning for typed sensors** must reference the actual
  runtime source, not per-route literals in `subscriber.rs`. The
  test `typed_sensor_staleness_matches_runtime_freshness_constants`
  catches drift between the wire-advertised window and runtime
  decay; new typed sensors must extend it.
- **The `eddi-Pending-forever` symptom** that triggered
  M-ACTUATED-RETRY: was caused by the eddi controller's old
  `parse_eddi` returning Stopped (sta=5 → Stopped pre-fix), so the
  `apply_dwell` Leave-arm fired (desired==current_mode), no
  actuation pushed, phase stuck at Pending. PR-EDDI-SENSORS-1
  fixed the parser; PR-ACT-RETRY-1 fixed the structural retry
  hole. The two together close the failure mode end-to-end.
