### A-46 — attempted fix reverted, closed note-only (no commit)

Attempted a no-import clamp in the evening-discharge branch; adversarial review proved the clamp was **vacuous**:

- Evening branch has `setpoint_target = min_pre.min(-export_power)` with `min_pre ∈ {10, -200}`, so the commanded setpoint is structurally ≤ +10 W. The clamp condition (setpoint > 0) could only fire for values in `(0, 10]`, and `prepare_setpoint` promotes any non-negative value to `IDLE_SETPOINT_W = 10` anyway. Output is bit-identical with and without the clamp.
- Both unit tests were vacuous: `a46_evening_discharge_clamps_to_no_import_...` asserted `<= 10` which holds either way; the knob-off test compared two identical outputs.
- The real scenario (7 kW Zappi exceeding battery+PV rate under `allow_battery_to_car=true`) is a physics/rate-limit issue at the inverter, not a setpoint-command bug. No controller-layer clamp fixes it.

Ledger entry rewritten with the root cause. Option (b) — early-return to idle when the knob is on and Zappi draws heavily — reverses the user's opt-in intent; it's a product-call, not a bug fix. Flagged for user clarification.
