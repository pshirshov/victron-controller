## User request

> Replace the brittle "limit grid export to current solar production when
> zappi is active" rule (`crates/core/src/controllers/setpoint.rs:617-637`)
> with a closed-loop controller that uses **compensated battery drain** as
> the feedback signal, accounting for two intentional grid-side loads (heat
> pump and cooker) which the operator metered specifically so they can be
> excluded from the "is the battery being drained for the EV?" question.

Locked design (agreed before the loop started):
- Compensated drain = `max(0, -battery_dc_power - heat_pump - cooker)`.
- Soft loop with asymmetric P controller: tighten when drain > threshold
  (default 1000 W), relax toward `-solar_export` when below.
- Hard safety clamp only in Fast mode + opt-in false (default 200 W).
- 4 new sensors (HP/cooker via zigbee2mqtt MQTT; two MPPT op-modes via
  Victron D-Bus, observability-only).
- 5 new knobs (threshold, relax_step, kp, target, hard_clamp).
- Folded the legacy `(2..8)` Soltaro carve-out into the unified loop.
- 23:55-00:00 Soltaro protection window untouched.

## Round-by-round defect highlights

### PR-ZD-3 round 1 — major control-law bug caught

The plan's relax formula `(prev + relax_step).max(-solar_export)` was
direction-asymmetric: only converged toward `-solar_export` from below.
Once the setpoint reached or exceeded `-solar_export` (e.g., after a
single tighten cycle drove it to idle 10 W), the relax branch would
walk the setpoint AWAY from the target indefinitely, permanently
disabling solar export.

The bug was in the plan, not just the executor's implementation. Round-1
adversarial review caught it specifically because the reviewer traced
the formula's behaviour from each initial-condition region. Fix:
bidirectional step-toward construction:

```rust
if prev < target { (prev + step).min(target) }
else { (prev - step).max(target) }
```

Three existing tests (15, 18, 21) had their expected values updated to
match the corrected gradual walk. Added `relaxes_setpoint_from_above_target_*`
as a regression guard.

This is the single most important catch of the session — without round-1
review, the soft loop would have shipped with a fundamental defect that
would have been observed in production within hours.

## Cross-cutting lessons

- **Plan formulae must be traced from every initial-condition region.**
  The PR-ZD-3 D01 bug existed because the plan said "relax toward
  -solar_export" and gave a formula that worked from one direction. A
  one-line addition of "verify the formula converges from any initial
  state" would have caught this at planning time.

- **Per-sensor range guards belong at the `apply_sensor_reading`
  chokepoint, not the dispatch layer.** PR-ZD-1's D01 fix added a
  helper next to the matching arms; future per-sensor sanity guards
  should mirror that placement.

- **`KnobValue::Int32` absence**: `target_w` (signed) routed through
  `KnobValue::Float` with `v.round() as i32` cast. The wire-format
  variant change was avoided as additive. If many more signed knobs
  arrive, a future PR should add `Int32`.

- **Helper deduplication only when behaviour can be proven identical.**
  PR-ZD-4's `compute_compensated_drain` helper extraction relied on
  PR-ZD-3's existing tests (15, 17-21, etc.) passing with unchanged
  expected values to prove the soft-loop refactor was a pure code-move.

## Constraints captured for future work

- The bidirectional step-toward formula is the canonical relax
  behaviour. Do not "simplify" back to one-direction max() — that's
  the original bug.
- Hard clamp is Fast-mode only by design. Eco/Eco+ rely on Zappi's
  CT self-modulation. Do not extend without explicit operator input.
- HP/cooker stale → 0 W in the loop (conservative; clamps tighter on
  dead bridge). Do not change to "use last value".
- The 23:55-00:00 Soltaro protection window MUST stay untouched —
  field-tested protection against a grid quirk.
- MPPT op-mode is observability-only. No control-loop coupling without
  a follow-up that explicitly relitigates the decision.
- `world.zappi_mode.target.value` (commanded), NOT
  `world.typed_sensors.zappi_state.value.zappi_mode` (readback) — the
  hard clamp arms predictively.
- The shared `compute_compensated_drain` helper is the single source
  of truth. Future formula changes update the helper, not duplicate
  inline.
