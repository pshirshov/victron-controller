## Milestones / PRs landed this session

### M-APP-UPTIME (commit `98c76c2`)

`scripts/install-victron.sh` chmod fix: `/data/rcS.local` was being
created by `touch` with 0644 perms; Venus's boot init runs the file as
a program (`test -x /data/rcS.local && /data/rcS.local`), so a freshly
installed file was silently skipped and the controller never came up
post-reboot. Fix: append `chmod +x /data/rcS.local` after the heredoc
in `install-victron.sh`.

App-uptime liveness sensor:

- `ControllerObservableId::AppUptimeS` (dotted name `controller.uptime-s`).
- Shared constants `APP_UPTIME_PUBLISH_PERIOD_S = 30` and
  `APP_UPTIME_EXPIRE_AFTER_S = 90` in `crates/shell/src/lib.rs` so
  publisher cadence and HA `expire_after` cannot drift.
- HA discovery entity: `device_class: duration`, `state_class:
  total_increasing`, `unit_of_measurement: s`,
  `expire_after: 90` — after two missed publishes (~60–90 s) HA
  marks the entity unavailable. That's the liveness signal.
- Publisher in `main.rs`: captures `process_start: Instant` as the
  first statement of `main`; spawns a 30 s ticker that publishes
  `process_start.elapsed().as_secs()` to
  `<topic_root>/controller/controller.uptime-s/state` via the MQTT
  client handle. `MissedTickBehavior::Skip`, 1 s publish timeout,
  `eprintln!` on failure (no `tracing!` to avoid feeding back through
  `MqttLogLayer` and amplifying a wedged broker).

Crash-restart story: no code change needed. svscan re-spawns the run
script within ~1 s on any non-zero exit; the boot-fix above is what
made that path actually reachable post-reboot.

Verification: `cargo test --workspace` (212 passed), clippy clean,
`tsc --noEmit -p web` clean.

### M-WSOC-TABLE / PR-WSOC-TABLE-1 (commit `7f779fa`)

Replaced the cascading rung ladder in
`crates/core/src/controllers/weather_soc.rs` with a 6×2 lookup table
indexed by (energy bucket, temperature column). Six energy buckets
(VerySunny / Sunny / Mid / Low / Dim / VeryDim) × two temperature
columns (warm > 12 °C / cold ≤ 12 °C) = 12 cells, each carrying 4
fields:

- `export_soc_threshold` (f64, %)
- `battery_soc_target` (f64, %)
- `discharge_soc_target` (f64, %)
- `extended` (bool)

`disable_night_grid_discharge` is **derived from `cell.extended`
before the override applies** — the rung-7 stacked override only
mutates `bat` + `ext`, leaving `dng` intact. This preserves the
cascade's behaviour in the override case verbatim.

New bucket-boundary knob `weathersoc_very_sunny_threshold` (default
67.5 kWh) replaces the hardcoded `1.5 × too_much` multiplier — full
11-layer CLAUDE.md registration walked.

The 12-cell value matrix is wire-modeled as a single nested
`WeatherSocTable` baboon type on `Knobs`. The 48 underlying
cell-knobs are NOT individually addressable: no flat `KnobId`
variants, no `apply_knob` arms, no HA discovery entries, no
`KnobsDefaultsConfig` fields, no flat `KNOB_SPEC` rows. Surfaced
read-only on the dashboard's Detail tab as a 6×3 widget (bucket
label + warm group + cold group, 4 sub-fields per cell). Per-cell
editing is a future PR.

Plan doc: `docs/drafts/20260504-1530-m-wsoc-table-plan.md`.
Memory: `~/.claude/projects/.../memory/project_weathersoc_table.md`.

Test count: 11 retained cascade-equivalence goldens + 19 new (12
cell-pinning + 3 override + 4 boundary) = 30 total in
`weather_soc.rs`. All four verification commands clean.
